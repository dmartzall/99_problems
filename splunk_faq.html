Terms and Definitions

Splunk - A platform for collecting, searching, monitoring, reporting and analyzing all of your real-time and historical machine data via a cloud service centrally and uniformly delivered

Report - A saved search that can be saved and/or be used to create alerts

Event - Anything that happens in the system

Alert - A defined event generated and sent to the ServiceNow platform containing technical information about a warning or critical event

Notification - A message sent to a person or team about an actionable alert

Index - The repository for data. When the Splunk platform indexes raw data, it transforms the data into searchable events.

Data Model - A type of knowledge object that applies an information structure to raw data, making it easier to use. Each data model represents a category of event data.

Source - A default field that identifies where the event originated. In the case of data monitored from files and directories, the source consists of the full pathname of the file or directory. In the case of a network-based source, the source field consists of the protocol and port, such as UDP:514.

Source Type -A default field that identifies the data structure of an event. A source type determines how Splunk Enterprise formats the data during the indexing process.  Example source types include access_combined and cisco_syslog.

Tag - A knowledge object that enables you to search for events that contain particular field values. You can assign one or more tags to any field/value combination, including event types, hosts, sources, and source types.

Account Issues

Q. I cannot log into Splunk.

A. Please ensure you have an account in Splunk and that you are using the correct credentials.  If the problem persists, please submit an ASKIT request. 

Q.  How do I gain access to Splunk?

A.  Submit a security request saying you need access to the Splunk platform

Q. What level of access do I need?

A. Access will depend on your role and your need.  The roles are broken out as follows:

  • MGIC users
    • app-splunk-admin - Splunk admins who will be responsible for maintaining the Splunk tool
    • app-splunk-dev - Consists of developers with access to all indexes not deemed IRM only data
    • app-splunk-infra - Consists of infrastructure engineers with access to all indexes not deemed IRM only data
    • app-splunk-support - Consists of lower tier support members with limited access to some indexes
    • app-splunk-security - Group for IRM with access to all indexes
  • Tavant users
    • app-splunk-tavant-dev - Tavant developer access group into Splunk
    • app-splunk-tavant-devops - Tavant devops access group into Splunk
    • app-splunk-tavant-prod-support - Tavant prod support access group into Splunk

General Issues

Q. Where is Splunk data saved?

A. Amazon Web Services (AWS) Data Centers: US (Virginia, Oregon, GovCloud)

Q. Can I get reports from Splunk?

A. Yes you can.  If you need assistance creating a report please submit an ASKIT request.

Q. Can Splunk generate alerts?

A. Yes it can. If you need an alert created please submit an ASKIT request.

Q. Can I create dashboards?

A. Yes you can, however depending on your permissions you may not be able to share them with team mates.  If you need a dashboard created for your team or require assistance please submit and ASKIT request.

Q. Which browsers are supported?

A. Chrome (latest), Firefox (latest), Internet Explorer 11 (Splunk Cloud does not support this browser in Compatibility Mode), and Safari (latest)

Q. Where can I find the Splunk Run Book created by Ahead?

A. MGIC_Splunk_Runbook

Q. Where can I find the support Wiki for Splunk?

A. https://tools.mgic.net/xwiki/bin/view/Tools/Splunk/

Search Issues

Q. Where can I find a quick reference for searches?

A. https://www.splunk.com/pdfs/solution-guides/splunk-quick-reference-guide.pdf

Q. Why can't I do real-time searches?

A. Real-time searches create a generally poor experience for all other users while a real-time search is active.  As such they are generally limited to admins and should only be used while needed.

Q. Why does my search take so long?

A. You likely have a large window setup in your timeframe for the search or you have created a search that is pulling back too many results.

Display Issues

Q. Why do my colors look off on the dashboard?

A.  Make sure that you are using a supported browser and if the dashboard was made with dark-mode off that your dark-mode is also disabled.

Q. Why can't I see a specific dashboard?

A. That dashboard may not have been shared to you and/or your role may not permit access to that index.

Q. Why does my display of Splunk look different than everyone else?

A. You may be using an incompatible browser.  Splunk supports: Chrome (latest), Firefox (latest), Internet Explorer 11 (Splunk Cloud does not support this browser in Compatibility Mode), and Safari (latest)

Log Issues

Q. Why can't I find my logs?

A. You may be searching in the wrong index or those logs may not have been setup for ingestion yet.  Try your search again without the index declared.  If that still doesn't work, please submit and ASKIT request.
